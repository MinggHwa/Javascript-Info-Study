### 2.2 코드 구조

- **문** : 문법구조와 명령어를 의미

### 2.3 엄격 모드

### 2.4 변수와 상수✅

#### 변수

> 변수 : 데이터를 저장할 때 쓰이는 '이름있는 저장소'.
> var, let, const

- let : 재할당 가능한 데이터 (변수)

```js
let num = 0;
num = 2;
```

- const : 재할당이 불가능한 데이터 (상수)

```js
const num = 0;
num = 2; // 오류 발생
```

- var : 오래된 키워드, 잘 사용하지 않음.
  스코프의 차이가 있음.

#### 변수 명명 규칙.

- 문자 숫자 기호`$` `_` 만 가능
- 첫 글자 숫자 안됨
- 예약어 안됨
- 대소문자 구별
- 엄격모드 없이는 꼭 정의 되어 있어야 함

### 2.5 자료형 ✅

#### 8가지 자료형

> JS는 기본 8가지 자료형이 있다.
> 숫자, Bigint, 문자, 불린, null, undefined, 객체, 심볼

- 숫자 : 정수 및 부동소수점 숫자
  `infinity`, `NaN`는 특수 숫자값이다.

- Bigint : 길이 제약 없이 정수를 나타낼 수 있다 정수리터럴 끝에 `n`을 붙이면 표현이 가능하다.

- 문자 : 큰따옴표, 작은따옴표, 백틱 을 사용하면 표현할 수 있다.

- 불린 : `true` or `false` 두가지 값

- null : 비어있는 값.

- nudefiend : 알수 없는 값

- 객체: 키-값 쌍의 집합

- 심볼: 고유하고 변경 불가능한 식별자

#### typeof 연산자

문법

> 연산자: typeof x
> 함수: typeof(x)

```js
typeof undefined; // "undefined"

typeof 0; // "number"

typeof 10n; // "bigint"

typeof true; // "boolean"

typeof "foo"; // "string"

typeof Symbol("id"); // "symbol"

typeof Math; // "object"  (1)

typeof null; // "object"  (2)

typeof alert; // "function"  (3)
```

1 - 메소드는 내장 객체
2 - 오류지만 하위 호환성 고료
3 - 함수도 객체임

### 2.10 if와 '?'를 사용한 조건 처리

#### if 문

- if(...)문은 괄호 안에 들어가는 조건을 평가
  참이면 코드 블록 실행
  거짓이면 else절 실행
- else if로 조건을 여러개 생성 가능

#### 조건부 연산자 ?

A ? B : C;
A의 조건이 참이면 B를 실행 거짓이면 C를 실행

다중으로 작성 가능하지만 가독성 저하.

### 2.11 논리 연산자

#### OR 연산자 ||

피연산자 중 하나라도 참이면 참을 반환.
평가 방식: 첫 번째 truthy 값을 반환하거나 모두 falsy면 마지막 값을 반환.

#### AND 연산자 &&

피연산자가 모두 참이어야 참을 반환.
평가 방식: 첫 번째 falsy 값을 반환하거나 모두 truthy면 마지막 값을 반환.

#### NOT 연산자 !

피연산자의 boolean 값을 반대로 반환.
평가 방식: 피연산자가 truthy이면 false를, falsy이면 true를 반환.

#### 다중 논리 연산자

여러 논리 연산자를 결합하여 사용 가능.
OR 연산자: 왼쪽에서 오른쪽으로 평가, 첫 번째 truthy 값을 반환.
AND 연산자: 왼쪽에서 오른쪽으로 평가, 첫 번째 falsy 값을 반환.

### 2.12 nullish 병합 연산자 '??'

null 또는 undefined 값을 처리하기 위해 사용되는 연산자.
문법: a ?? b 형태로 사용, a가 null 또는 undefined이면 b를 반환, 그렇지 않으면 a를 반환.

#### || 연산자와의 차이점

- || 연산자: 첫 번째 truthy 값을 반환.
- ?? 연산자: 첫 번째 null 또는 undefined가 아닌 값을 반환.

```js
let height = 0;
alert(height || 100); // 100
alert(height ?? 100); // 0
```

#### 연산자 우선순위

우선순위: ??는 ||, &&보다 낮고, =보다 높다.

#### 사용 시 주의사항

명확한 사용: ??는 null 병합 전용으로 사용. 다른 논리 연산자와 함께 사용할 때는 괄호로 묶어 사용.
오류 방지: ??는 ||나 &&와 함께 사용할 수 없으며, 이를 혼용하면 문법 오류 발생.

### 2.13 while과 for 반복문

#### 반복문

> 조건이 참인 동안 코드 블록을 반복해서 실행.

#### while

> 구문: while (조건) { ... }
> 조건이 참일 동안 반복 실행.

#### do..while 문

> 구문: do { ... } while (조건);
> 코드 블록을 먼저 실행한 후 조건을 평가.

#### for 문

> 구문: for (시작; 조건; 반복) { ... }
> 반복 변수를 초기화, 조건 평가, 반복문 실행 후 변수를 업데이트.

#### 반복문 제어

break: 반복문을 완전히 종료.
continue: 현재 반복을 종료하고 다음 반복으로 넘어감.

#### 레이블(Label)

레이블을 사용해 다중 반복문에서 원하는 반복문을 제어할 수 있음.

```js
outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) break outer;
    alert(i + "," + j);
  }
}
```

### 2.14 switch문

> switch (표현식) { case 값1: 코드 블록1; break; case 값2: 코드 블록2; break; ... default: 코드 블록; }

- 표현식의 결과를 case 값과 비교하여 일치하는 코드 블록을 실행.
- break 문으로 switch 블록을 종료하고 다음 코드를 실행.
- default는 모든 case가 일치하지 않을 때 실행됨.

```js
et fruit = 'apple';
switch (fruit) {
  case 'banana':
    alert('바나나');
    break;
  case 'apple':
    alert('사과');
    break;
  default:
    alert('알 수 없음');
}
```

### 2.15 함수

> 함수: 코드 블록을 정의하여 특정 작업을 수행하게 할 수 있는 구조.
> function 함수명(매개변수1, 매개변수2, ...) { ... }

함수는 매개변수를 받아 작업을 수행하고, return 문으로 결과를 반환.

```js
function sum(a, b) {
  return a + b;
}
alert(sum(2, 3)); // 5
```

### 2.16 함수 표현식✅

> let 함수명 = function(매개변수1, 매개변수2, ...) { ... };

```js
let subtract = function (a, b) {
  return a - b;
};
alert(subtract(5, 2)); // 3
```

- 함수 선언: 함수 이름으로 호출 가능
- 함수 표현식: 함수를 변수에 할당하여 호출

함수 표현식은 변수에 할당되며, 변수 이름으로 호출됨.
익명 함수: 이름이 없는 함수 표현식

### 2.17 화살표 함수 기본

> (매개변수1, 매개변수2, ...) => { ... }

```js
const add = (a, b) => a + b;
```

- 함수 표현식을 간결하게 작성하는 방법.
- return 문이 필요한 경우, 중괄호 {}와 함께 사용.
  간단한 표현식인 경우 중괄호와 return 생략 가능.
